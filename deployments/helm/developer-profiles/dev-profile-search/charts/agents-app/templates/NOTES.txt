=====================================================================
  QUICK START
=====================================================================

1. Get the VSS UI URL:
   {{- if .Values.vssUi.enabled }}
   export SERVICE_IP=$(kubectl get svc --namespace {{ .Release.Namespace }} {{ include "agents.fullname" . }}-vss-ui -o jsonpath='{.status.loadBalancer.ingress[0].ip}')
   echo "VSS UI: http://$SERVICE_IP:{{ .Values.vssUi.port }}"
   {{- end }}

2. Port-forward to access services locally:
   {{- if .Values.vssUi.enabled }}
   kubectl port-forward svc/{{ include "agents.fullname" . }}-vss-ui {{ .Values.vssUi.port }}:{{ .Values.vssUi.port }} --namespace {{ .Release.Namespace }}
   {{- end }}
   {{- if .Values.vssAgent.enabled }}
   kubectl port-forward svc/{{ include "agents.fullname" . }}-vss-agent {{ .Values.vssAgent.port }}:{{ .Values.vssAgent.port }} --namespace {{ .Release.Namespace }}
   {{- end }}

3. Check pod status:
   kubectl get pods --namespace {{ .Release.Namespace }} -l "app.kubernetes.io/instance={{ .Release.Name }}"

=====================================================================
  PREREQUISITES
=====================================================================

Ensure the following services are running before using VSS Agents:

1. VST MCP Service (for video management)
2. NIM Services (Nemotron, Cosmos) for LLM capabilities
3. Elasticsearch/Kibana for dashboard functionality
4. MDX Web API for alerts integration
